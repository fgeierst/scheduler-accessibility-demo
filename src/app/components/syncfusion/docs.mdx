## Keyboard Functionality

Two implementation-level flags matter for keyboard support:

- `allowKeyboardInteraction`: enables Scheduler keyboard interaction (`true` by default).
- `allowClipboard`: enables cut, copy, and paste shortcuts for appointments, but only works correctly when `allowKeyboardInteraction` is enabled.

| Shortcut                                         | Action                                                                              | Status                                                     |
| ------------------------------------------------ | ----------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| <kbd>Alt</kbd> + <kbd>J</kbd>                    | Focus the Scheduler element                                                         | Documented by Syncfusion; application-level focus shortcut |
| <kbd>Tab</kbd>                                   | Move focus from the header bar into visible event elements, then out of the control | Documented by Syncfusion                                   |
| <kbd>Shift</kbd> + <kbd>Tab</kbd>                | Reverse the tab order through events and header items                               | Documented by Syncfusion                                   |
| <kbd>Enter</kbd>                                 | Open quick info on selected cells or events                                         | Documented by Syncfusion                                   |
| <kbd>Escape</kbd>                                | Close an open popup                                                                 | Documented by Syncfusion                                   |
| <kbd>Arrow Keys</kbd>                            | Move to adjacent cells in the requested direction                                   | Documented by Syncfusion                                   |
| <kbd>Shift</kbd> + <kbd>Arrow Keys</kbd>         | Extend multi-cell selection                                                         | Documented by Syncfusion                                   |
| <kbd>Delete</kbd>                                | Delete selected events                                                              | Documented by Syncfusion                                   |
| <kbd>Ctrl</kbd> + click                          | Multi-select events                                                                 | Documented by Syncfusion                                   |
| <kbd>Alt</kbd> + <kbd>1</kbd> to <kbd>6</kbd>    | Switch between Scheduler views                                                      | Documented by Syncfusion                                   |

### Notes

- Syncfusion explicitly says `Alt + J` must be implemented at the application layer to move focus into the Scheduler.
- Clipboard shortcuts are not universal by default; they depend on `allowClipboard` plus `allowKeyboardInteraction`.
- Syncfusion also documents pointer and touch interactions separately, including single-click selection, double-click editing, drag-and-drop, resizing, and swipe navigation on touch devices.

## Semantic Structure

Syncfusion documents the Scheduler as a WAI-ARIA-based web component with roles, states, and properties applied to its interactive elements.

### Documented Roles And Attributes

| Documented semantic                                                           | Meaning                                                    |
| ----------------------------------------------------------------------------- | ---------------------------------------------------------- |
| `role="main"` on the Scheduler parent                                         | Marks the Scheduler as the component’s main content area   |
| `role="button"` on appointments                                               | Exposes appointments as interactive controls               |
| `aria-label` on the Scheduler root                                            | Announces the current date and updates as the date changes |
| `aria-label` on navigation controls, date-range header text, and appointments | Provides accessible names for header and event controls    |
| `aria-labelledby`                                                             | Associates the editor dialog with its title                |
| `aria-describedby`                                                            | Associates the editor dialog with descriptive content      |
| `aria-disabled` on appointments                                               | Conveys disabled event state to assistive technologies     |

## Features

- Details view: Syncfusion Scheduler includes a built-in quick info popup for selected events when quick info is enabled.
- Add view: Syncfusion Scheduler includes a built-in editor window for creating events.

## Accessibility Claims In Vendor Docs

Syncfusion states that the Angular Scheduler supports:

- WCAG 2.2
- Section 508
- screen readers
- RTL
- color contrast
- mobile devices
- keyboard navigation
- automated accessibility checking with accessibility-checker and axe-core
